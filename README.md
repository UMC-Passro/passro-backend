# Git Convention

## 커밋 컨벤션

커밋 메시지는 아래 형식을 따릅니다.

```text
<type>: <description>
```

| Type | 설명 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `style` | 코드 포맷팅 |
| `refactor` | 코드 리팩토링 |
| `chore` | 기타 설정 변경 |

## PR / 이슈 템플릿

```markdown
## 설명

어떤 내용에 대해 이슈를 발행하는지 작성합니다.

## Todos

- [ ] 해결해야 하는 구체적인 내용을 작성합니다.
```

## Code Review 시 확인할 사항

- 프로젝트 컨벤션을 지켰는가?
- 불필요한 수정을 하지 않았는가?
- 불필요한 자원 소모를 요구하지는 않는가?
- 예외가 발생할 만한 곳의 예외가 처리되지 않았는가?

## Pull Request Merge

- Merge하기 전 유닛 테스트와 통합 테스트를 통과해야 합니다.
- 상호 코드 리뷰를 권장합니다. 개발 속도를 고려해 CodeRabbit을 도입하고, 코드 리뷰 내용을 해결한 뒤 Merge합니다.
- Merge할 때는 Squash Merge를 권장합니다.
